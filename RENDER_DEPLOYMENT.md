# Render Deployment Guide

## Prerequisites

1. GitHub account with your repository
2. Render account (sign up at https://render.com)
3. MongoDB Atlas account for database (https://www.mongodb.com/cloud/atlas)

## Step 1: Prepare MongoDB Atlas

1. Create a free MongoDB Atlas cluster
2. Create a database user with password
3. Whitelist all IPs (0.0.0.0/0) for Render access
4. Get your connection string (looks like: `mongodb+srv://username:password@cluster.mongodb.net/georgetown-traffic`)

## Step 2: Deploy to Render

### Option A: Using render.yaml (Recommended)

1. Push your code to GitHub
2. Go to Render Dashboard
3. Click "New" → "Blueprint"
4. Connect your GitHub repository
5. Render will detect `render.yaml` and create both services
6. Set the following environment variables:

**Backend Service:**
- `MONGODB_URI`: Your MongoDB Atlas connection string
- `CORS_ORIGIN`: Your frontend URL (will be provided after frontend deploys)
- `PYTHON_AI_SERVICE_URL`: Leave empty for now (optional Python service)

**Frontend Service:**
- `VITE_API_URL`: Your backend URL (will be provided after backend deploys)

7. Click "Apply" to deploy both services

### Option B: Manual Deployment

#### Deploy Backend:

1. Go to Render Dashboard
2. Click "New" → "Web Service"
3. Connect your GitHub repository
4. Configure:
   - **Name**: georgetown-traffic-backend
   - **Region**: Oregon (or closest to you)
   - **Branch**: master (or main)
   - **Root Directory**: backend
   - **Environment**: Node
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Plan**: Starter (or Free)

5. Add Environment Variables:
   ```
   NODE_ENV=production
   PORT=5000
   MONGODB_URI=<your-mongodb-atlas-uri>
   JWT_SECRET=<auto-generated-by-render>
   JWT_EXPIRES_IN=7d
   CORS_ORIGIN=<your-frontend-url>
   MAX_REQUEST_SIZE=10mb
   REQUEST_TIMEOUT=30000
   RATE_LIMIT_WINDOW_MS=900000
   RATE_LIMIT_MAX_REQUESTS=100
   ```

6. Click "Create Web Service"

#### Deploy Frontend:

1. Click "New" → "Static Site"
2. Connect your GitHub repository
3. Configure:
   - **Name**: georgetown-traffic-frontend
   - **Branch**: master (or main)
   - **Root Directory**: frontend
   - **Build Command**: `npm install && npm run build`
   - **Publish Directory**: dist

4. Add Environment Variable:
   ```
   VITE_API_URL=<your-backend-url>
   ```

5. Click "Create Static Site"

## Step 3: Update CORS Settings

After both services are deployed:

1. Copy your frontend URL (e.g., `https://georgetown-traffic-frontend.onrender.com`)
2. Go to backend service settings
3. Update `CORS_ORIGIN` environment variable with your frontend URL
4. Backend will automatically redeploy

## Step 4: Initialize Database

1. Create an admin user by making a POST request to your backend:

```bash
curl -X POST https://your-backend-url.onrender.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "email": "admin@example.com",
    "password": "YourSecurePassword123!",
    "role": "admin"
  }'
```

2. Import Georgetown network data (if you have it locally):
   - You'll need to run the import script from your local machine pointing to the production database
   - Or manually upload via the admin panel once logged in

## Step 5: Verify Deployment

1. Visit your frontend URL
2. Check health endpoint: `https://your-backend-url.onrender.com/health`
3. Try logging in with your admin credentials
4. Check the logs in Render dashboard for any errors

## Environment Variables Reference

### Backend Required Variables:
- `NODE_ENV`: production
- `PORT`: 5000 (Render sets this automatically)
- `MONGODB_URI`: MongoDB connection string
- `JWT_SECRET`: Secret for JWT tokens (auto-generated)
- `CORS_ORIGIN`: Frontend URL(s), comma-separated

### Backend Optional Variables:
- `JWT_EXPIRES_IN`: Token expiration (default: 7d)
- `MAX_REQUEST_SIZE`: Max request body size (default: 10mb)
- `REQUEST_TIMEOUT`: Request timeout in ms (default: 30000)
- `RATE_LIMIT_WINDOW_MS`: Rate limit window (default: 900000)
- `RATE_LIMIT_MAX_REQUESTS`: Max requests per window (default: 100)
- `PYTHON_AI_SERVICE_URL`: Python AI service URL (optional)

### Frontend Required Variables:
- `VITE_API_URL`: Backend API URL

## Troubleshooting

### Backend won't start:
- Check MongoDB connection string is correct
- Verify all required environment variables are set
- Check logs in Render dashboard

### Frontend can't connect to backend:
- Verify `VITE_API_URL` is set correctly
- Check CORS settings on backend
- Ensure backend is running (check health endpoint)

### Database connection issues:
- Verify MongoDB Atlas IP whitelist includes 0.0.0.0/0
- Check database user has correct permissions
- Test connection string locally first

## Free Tier Limitations

Render free tier includes:
- Services spin down after 15 minutes of inactivity
- First request after spin-down takes 30-60 seconds
- 750 hours/month of runtime
- Consider upgrading to Starter plan ($7/month) for always-on service

## Monitoring

- View logs in Render dashboard
- Set up alerts for service failures
- Monitor MongoDB Atlas metrics
- Use Render's built-in metrics for request rates and response times

## Updating Your Deployment

Render automatically deploys when you push to your connected branch:

```bash
git add .
git commit -m "Update feature"
git push origin master
```

Or manually trigger a deploy from the Render dashboard.

## Security Notes

1. Never commit `.env` files
2. Use strong passwords for MongoDB
3. Rotate JWT_SECRET periodically
4. Keep dependencies updated
5. Monitor logs for suspicious activity
6. Enable 2FA on Render and MongoDB Atlas accounts
