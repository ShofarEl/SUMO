## Quick Deploy to Vercel

### Option 1: Web Interface (Easiest)
1. Push code to GitHub
2. Go to https://vercel.com/new
3. Import your repository
4. Click "Deploy" (auto-detects config)
5. Done! Get your URL

### Option 2: CLI
```bash
npm install -g vercel
vercel login
cd frontend
vercel --prod
```

### Option 3: Use Helper Script
```bash
deploy-to-vercel.bat
```

## After Deployment

### Update Backend CORS (CRITICAL)

1. Get your Vercel URL (e.g., `https://your-project.vercel.app`)
2. Go to https://dashboard.render.com
3. Select service: **sumo-d68k**
4. Go to **Environment** tab
5. Update `CORS_ORIGIN`:
   ```
   CORS_ORIGIN=https://your-project.vercel.app
   ```
6. Save (backend auto-redeploys in ~2 minutes)

**See UPDATE_CORS.md for detailed instructions**

## Costs

### Current Setup (Free Tier)
- **Vercel**: Free (Hobby plan)
  - Unlimited deployments
  - 100 GB bandwidth/month
  - Automatic HTTPS
  
- **Render**: Free tier
  - Spins down after 15 min inactivity
  - 750 hours/month
  - First request after spin-down: 30-60s
  
- **MongoDB Atlas**: Free tier
  - 512 MB storage
  - Shared cluster

**Total: $0/month**

### Recommended Production Setup
- **Vercel**: Free (sufficient for most cases)
- **Render Starter**: $7/month (always-on, no spin-down)
- **MongoDB Atlas M10**: $57/month (dedicated, 2GB RAM)

**Total: ~$64/month**
