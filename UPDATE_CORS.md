# Update Backend CORS for Vercel

After deploying your frontend to Vercel, you need to update the backend CORS settings.

## Step 1: Get Your Vercel URL

After deploying to Vercel, you'll receive a URL like:
- `https://georgetown-traffic-ai.vercel.app`
- Or your custom domain

## Step 2: Update Render Environment Variables

1. Go to https://dashboard.render.com
2. Select your service: **sumo-d68k**
3. Click on **Environment** in the left sidebar
4. Find the `CORS_ORIGIN` variable
5. Update it to include your Vercel URL:

```
CORS_ORIGIN=https://your-project.vercel.app
```

Or if you have multiple domains:
```
CORS_ORIGIN=https://your-project.vercel.app,https://custom-domain.com
```

6. Click **Save Changes**
7. Your backend will automatically redeploy (takes ~2 minutes)

## Step 3: Verify CORS is Working

1. Open your Vercel frontend URL
2. Open browser DevTools (F12)
3. Go to Console tab
4. Try logging in or making an API call
5. Check for CORS errors

### If you see CORS errors:

```
Access to XMLHttpRequest at 'https://sumo-d68k.onrender.com/api/...' 
from origin 'https://your-project.vercel.app' has been blocked by CORS policy
```

**Solution:**
- Double-check the CORS_ORIGIN value on Render
- Make sure there are no trailing slashes
- Verify the URL matches exactly (including https://)
- Wait for backend to finish redeploying

## Testing CORS

You can test if CORS is configured correctly:

```bash
# Test from command line
curl -H "Origin: https://your-project.vercel.app" \
     -H "Access-Control-Request-Method: POST" \
     -H "Access-Control-Request-Headers: Content-Type" \
     -X OPTIONS \
     https://sumo-d68k.onrender.com/api/auth/login -v
```

Look for these headers in the response:
```
Access-Control-Allow-Origin: https://your-project.vercel.app
Access-Control-Allow-Methods: GET, POST, PUT, DELETE, PATCH, OPTIONS
Access-Control-Allow-Headers: Content-Type, Authorization
```

## Common Issues

### Issue 1: Multiple Origins Not Working

If you have multiple domains, separate them with commas (no spaces):
```
CORS_ORIGIN=https://app1.vercel.app,https://app2.vercel.app
```

### Issue 2: Wildcard Not Working

The backend doesn't support wildcard origins for security. You must specify exact domains.

### Issue 3: Backend Not Redeploying

After changing environment variables:
1. Check the "Events" tab on Render
2. You should see "Deploy triggered by config change"
3. Wait for "Deploy live" status
4. Usually takes 1-2 minutes

### Issue 4: Still Getting CORS Errors

1. Clear browser cache
2. Hard refresh (Ctrl+Shift+R or Cmd+Shift+R)
3. Try in incognito/private window
4. Check backend logs on Render for errors
